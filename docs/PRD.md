# BackupBeacon 产品需求文档（PRD）

## 1. 背景

团队同时维护多个项目数据库，需要稳定、自动化、可审计的备份体系，并具备异地冗余能力。人工备份方式存在执行不及时、易遗漏、难追踪等问题。

## 2. 产品目标

提供一个集中式内网备份平台，用户完成一次连接与策略配置后，系统即可按计划自动执行备份并记录完整执行轨迹。

## 3. 当前范围

### 3.1 包含范围

1. 多连接管理（首期支持 MySQL/PostgreSQL）
2. 备份策略管理
   - 全量备份
   - 增量/追加备份（按数据库引擎能力实现）
3. 多调度方式
   - 固定间隔
   - Cron 表达式
4. 备份目标同步
5. 任务历史与执行日志
6. 备份健康告警（失败/重试/超时）

### 3.2 不包含范围（当前阶段）

1. 恢复/回档流程
2. 公网部署能力
3. 内置鉴权与 RBAC

## 4. 核心用户故事

1. 作为运维人员，我可以新增多个数据库连接并验证连通性。
2. 作为运维人员，我可以按策略配置需要备份的库/表范围。
3. 作为运维人员，我可以设置固定间隔或 Cron 调度规则。
4. 作为运维人员，我可以配置异地存储目标与保留策略。
5. 作为运维人员，我可以查看任务执行历史并及时收到失败告警。

## 5. 功能需求

### 5.1 连接管理

- 支持数据库连接的新增、编辑、删除
- 保存前支持连接测试
- 凭证加密存储

### 5.2 备份策略

- 支持全量/增量模式选择
- 支持库级/表级范围配置
- 支持压缩、加密开关

### 5.3 调度系统

- 支持间隔调度与 Cron 调度
- 支持服务重启后的漏执行策略
- 支持单数据源并发控制

### 5.4 执行流水线

1. 读取策略
2. 获取源库执行锁（可选）
3. 执行导出
4. 压缩并计算校验
5. 上传异地对象存储
6. 持久化元数据记录
7. 产出成功/失败事件

### 5.5 监控与告警

- 任务状态时间线
- 失败原因记录
- 通知渠道（Telegram/Webhook）

## 6. 非功能需求

- 通过 Docker Compose 在内网部署
- 服务重启后调度配置不丢失
- 备份文件必须可做完整性校验
- UI 不展示明文密钥
- 最小网络暴露面

## 7. 里程碑

### v1

- 多数据库连接管理
- 全量备份能力
- 调度执行与备份上传
- 日志与告警

### v2

- 表级策略增强
- 按引擎优化增量机制
- 更细粒度保留策略管理

## 8. 验收标准（v1）

1. 支持配置不少于 5 个数据库连接。
2. 支持配置不少于 10 条备份策略。
3. 备份产物可成功上传到已配置异地对象存储。
4. 任务失败后 1 分钟内触发告警。
5. 核心服务可通过 `docker compose up -d` 启动。
