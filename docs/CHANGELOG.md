# BackupBeacon 改动清单

## 2026-03-03（v1 交互与可移植性优化，已拍板）

状态：P0 已完成，P1 已完成

### P0（本轮必做）

- [x] 目标目录提示改造为动态通用文案，不写死宿主机路径。
- [x] 后端新增路径配置接口，前端动态读取允许根目录（来自 `BACKUPBEACON_FS_ALLOWED_ROOTS`）。
- [x] 增加可选路径映射配置（`BACKUPBEACON_PATH_DISPLAY_MAP`），用于展示“容器路径 -> 宿主机路径”。
- [x] 立即备份按钮增加 loading 过渡、禁用重复点击、状态提示（排队/执行中/成功/失败）。

### P1（紧随其后）

- [x] 连接管理新增“描述/备注”字段（`description`）。
- [x] 连接列表新增“上次备份时间”字段（`lastBackupTime`，无记录显示 `-`）。

### 设计约束

- 校验与执行统一基于容器内路径（如 `/backup-target/mysql`）。
- 宿主机路径仅用于展示，不参与核心逻辑判断。
- 未配置 `BACKUPBEACON_PATH_DISPLAY_MAP` 时，页面仅显示容器路径提示。

### 验收口径

- 跨服务器部署时无需修改前端目录说明文案。
- 立即备份在大库场景下有明确执行反馈，不再出现“像卡住”的感知。
- 连接列表可直接识别用途（描述）和最近备份情况（上次备份时间）。
