# BackupBeacon 架构设计

## 1. 运行拓扑

- `bb-web`：前端界面与 API 网关入口
- `bb-api`：配置管理与任务编排 API
- `bb-scheduler`：调度引擎
- `bb-worker`：备份执行器
- `bb-meta-db`：元数据存储（PostgreSQL）
- `bb-object`（可选）：本地 S3 兼容目标（MinIO）

所有服务运行在内网 Docker 网络中。

## 2. 数据流

1. 用户在 Web 页面配置连接与备份策略。
2. API 将加密配置写入元数据数据库。
3. 调度器触发到期任务。
4. Worker 按数据库引擎执行导出命令。
5. 产物压缩、校验后上传对象存储。
6. 保存执行元数据与日志。
7. 告警模块对失败/超时任务发送通知。

## 3. 存储策略

- Worker 容器卷用于本地临时落盘（内网服务器）
- 异地目标通过 S3 兼容协议写入
- 保留策略清理由定时任务执行

## 4. 安全基线（不做应用鉴权前提）

- 仅允许内网部署
- 反向代理 IP 白名单
- 防火墙网段白名单
- 服务端口不直接暴露公网
- 凭证加密存储

## 5. 可靠性设计

- 指数退避重试
- 调度窗口幂等键
- 每次执行使用元数据事务包裹
- 所有容器提供健康检查
